## 1. 사용자 범위

| 사용자 | 설명 |
|---|---|
| 운영 관리자(3PL사업팀, 사업관리팀, BTF) | 인트라에서 `정적` 미션 캠페인을 등록·관리하는 내부 관리자 |

---

## 2. 캠페인 상태 정의

| 상태 | 설명 |
|---|---|
| 예정 | 등록 완료 후 시작일 전 |
| 진행 | 시작일 도달 시 자동 전환 |
| 완료 | 종료일 경과 시 자동 전환 |
| 중단 | 관리자 `OFF` 처리 |

### 상태 변경 기준

| 현재 상태 | 조건 | 변경 후 상태 | 비고 |
|---|---|---|---|
| 예정 | 시작일 도달 | 진행 | 시스템 자동 변경 |
| 진행 | 종료일 경과 | 완료 | 시스템 자동 변경 |
| 예정・진행 | 관리자 `OFF` | 중단 | 수동 |
| 중단 | 관리자 `ON` + 현재 시각 < 시작일 | 예정 | 아직 시작 전이므로 **예정**으로 복구 |
| 중단 | 관리자 `ON` + 시작일 ≤ 현재 시각 ≤ 종료일 | 진행 | 기간 내이므로 즉시 **진행**으로 복구 |
| 중단 | 종료일 경과 | 완료 | 자동, `ON` 불가 |

## 3. LNB 메뉴 구조

| depth 1 | depth 2 | depth 3 | 진입 방식 |
|---|---|---|---|
| 캠페인 관리 | 미션 관리 | 미션 목록 | LNB 클릭 |
| 캠페인 관리 | 미션 관리 | 미션 추가 | 미션 목록 내 + 추가 버튼 클릭 |

- 추후 프로모션·리워드 메뉴 추가 예정
- 미션 추가는 LNB에 직접 노출되지 않으며 **미션 목록에서만 진입** 가능

## 4. 기능 요구사항

### 4-1. 미션 관리 목록

> 등록된 미션을 1) **조회·ON/OFF 제어**, 2) **새 미션을 추가**할 수 있는 화면

#### 검색 조건

| 항목 | 설명 | 입력 형태 | 비고 |
|---|---|---|---|
| 상태 | 전체 / 예정 / 진행 / 완료 / 중단 | 드롭다운 |  |
| 목적 | 전체 / 전환 / 활성화 | 드롭다운 |  |
| 적용대상 유형 | 전체 / 일반기사(프렌즈) / NVP지점기사 / ~~벤더기사~~ | 드롭다운 | 벤더기사 P2(추후) |
| 미션명 | 미션명 키워드 검색 | 텍스트 |  |
| 검색 버튼 | - | 버튼 | 설정조건에 맞는 목록 조회 |
| 초기화 버튼 | - | 버튼 | 검색 조건 전체 초기화 클릭 시 전체 목록 조회 |

#### 미션 목록 항목

| 항목 | 설명 | 입력 형태 | 비고 |
|---|---|---|---|

| 목적 | 전환 / 활성화 | 텍스트(읽기전용) |  |
| 기간 | 시작일 ~ 종료일 | 텍스트(읽기전용) |  |
| 적용대상 유형 | 일반기사(프렌즈) / NVP지점기사 / ~~벤더기사~~ | 텍스트(읽기전용) | 벤더기사 P2(추후) |
| 지역・ZONE | 미션 적용 지역/ZONE | 텍스트(읽기전용) | 예: 미션적용지역: …, ZONE: … |
| 소진액 | 현재까지 지급된 총 금액 | 텍스트(읽기전용) |  |

| 남은 한도 | 잔여 예산 금액 + 잔여 비율 | 텍스트 + progress bar | 예: 4,750,000원 / 59% 남음 |
| 상태 | 예정 / 진행 / 완료 / 중단 | 뱃지 | 예정: 회색 / 진행: 초록 / 완료: 회색 / 중단: 레드 |

| 총 건수 | 조회된 미션 수 | 텍스트(읽기전용) | 목록 우측 하단 |
| + 추가 | 미션 추가 화면 진입 | 버튼 | 목록 하단 좌측 |

### 4-2. 미션 추가

#### 기본 정보

| 항목 | 설명 | 필수 | 입력 형태 | 동작 및 Validation |
|---|---|---|---|---|
| 미션명 | 미션 이름 | O | 텍스트 | 최대 50자 |
| 미션 설명 | 미션에 대한 설명 | X | 텍스트 | 최대 500자 |
| 목적 | 캠페인 운영 목적(전환/활성화) | O | Radio | 목적에 따라 지급 구조/조건이 달라짐 |
| 지급 구조 | 단일 목표형/단계형 | O | Radio | 전환 목적 시 미노출, 활성화 목적 시 선택 |

#### 적용 조건(요약)

- 미션 오픈일: **오픈일 ≥ 오늘**
- 기간 설정: **시작일 ≥ 오늘**, **종료일 > 시작일**
- 시간대 설정: 활성화 목적만, 다중 구간/중복 불가/미설정 시 전일
- 화주사: 전체/로컬/프랜차이즈/플랫폼(프랜차이즈 선택 시 리스트/세부)
- 미션 공개일: **공개일 ≤ 시작일**
- 예산한도: **0원 초과**, 지급 금액 입력 완료 시 `최대 참여 가능 인원 N명 · 1인당 지급 N원` 안내(단계형은 최고 단계 금액 기준)
- 가입 후 유효 일수: 전환 목적, **0 초과**
- 적용대상 유형: 1개 이상
- 라이더 활동 수준: 프렌즈 선택 시(선택)
- ZONE: NVP 선택 시(전체 토글 또는 개별 선택)
- 라이더 활동 지역 / 미션 대상 지역: 프렌즈 선택 시(전체지역 토글 또는 직접 선택)

#### 달성 조건(요약)

- 전환: 가입 후 유효 일수 내 목표 건수 달성 시 1회 지급 + 1회 지급 금액
- 활성화 단계형/단일형: 목표/지급 금액 규칙(단계 오름차순, 최대 10단계 등)

#### 공통 미션 운영 기준(전환/활성화 모두 노출)

| 항목 | 설명 | 필수 | 입력 형태 | 동작 및 Validation |
|---|---|---|---|---|
| 휴무 조건 | 미션 기간 내 최대 휴무 횟수<br>※ 하루 중 부릉플러스 앱 → 오더받기 ON 상태가 한 번도 없는 날을 휴무로 산정 | X | 숫자(회) | 0 이상 정수 |
| 취소건수 조건 | 미션 기간 내 취소 건수<br>※ 라이더가 직접 배차 취소한 건에 한정 | X | 숫자(건) | 0 이상 정수 |

#### 저장/조회

- 저장 성공: 동일 페이지 **조회 모드** 전환 + **예산 한도/남은 한도(프로그레스)** 노출
- 저장 실패: 사유 모달

#### 수정(핵심 변경)

- **미션 오픈일이 도래하지 않은 `예정` 상태**인 미션은 **모든 항목 수정 허용**
- 진행/완료/중단은 내용 수정 제한
- 예정/진행은 ON/OFF로 운영 중단/재개 가능

## 5. M캐시 지급 처리 플로우(요약)

- 참여 확정: 미션 기간 내 배송 1건 이상 수행 시 실시간 카운트
- 최대 참여 인원: `1인당 예산 ÷ 최고 지급 금액`(단계형은 최고 단계 금액)
- 정원 도달 시 즉시 미션 종료(진행→완료)
- 지급 처리: 미션 종료 다음날 배치 기반 후정산

### 선택 조건(설정된 경우에만 검증)

| 조건 | 기준 | 미충족 시 처리 |
|---|---|---|
| 휴무 조건 | 미션 기간 내 총 휴무 횟수 > 설정값 | 지급 제외 |
| 취소건수 조건 | 미션 기간 내 총 취소 건수 > 설정값 | 지급 제외 |

## 6. 라이더 유형 세그먼트 산정 기준(프렌즈 한정)

- 산정 시점: 미션 시작일 기준 직전 2주
- 대상 라이더: 재직/부릉프렌즈/첫 수행 완료/직전 2주 내 배달 완료 1건 이상 등(원문 표 기준)
- 분류: SLA(고/저) × 물량(고/저) 조합 4분류
- 처리: 시작 시점 스냅샷 고정, 직전 2주 이력 없으면 참여 대상 제외
