## 1. 사용자 범위

| 사용자 | 설명 |
|---|---|
| 운영 관리자(3PL사업팀, 사업관리팀, BTF) | 인트라에서 `정적` 미션 캠페인을 등록·관리하는 내부 관리자 |

---

## 2. 캠페인 상태 정의

| 상태 | 설명 |
|---|---|
| 예정 | 등록 완료 후 시작일 전 |
| 진행 | 시작일 도달 시 자동 전환 |
| 완료 | 종료일 경과 시 자동 전환 |
| 중단 | 관리자 `OFF` 처리 |

### 상태 변경 기준

| 현재 상태 | 조건 | 변경 후 상태 | 비고 |
|---|---|---|---|
| 예정 | 시작일 도달 | 진행 | 시스템 자동 변경 |
| 진행 | 종료일 경과 | 완료 | 시스템 자동 변경 |
| 예정・진행 | 관리자 `OFF` | 중단 | 수동 |
| 중단 | 관리자 `ON` + 현재 시각 < 시작일 | 예정 | 아직 시작 전이므로 **예정**으로 복구 |
| 중단 | 관리자 `ON` + 시작일 ≤ 현재 시각 ≤ 종료일 | 진행 | 기간 내이므로 즉시 **진행**으로 복구 |
| 중단 | 종료일 경과 | 완료 | 자동, `ON` 불가 |

## 3. LNB 메뉴 구조

| depth 1 | depth 2 | depth 3 | 진입 방식 |
|---|---|---|---|
| 캠페인 관리 | 미션 관리 | 미션 목록 | LNB 클릭 |
| 캠페인 관리 | 미션 관리 | 미션 추가 | 미션 목록 내 + 추가 버튼 클릭 |

- 추후 프로모션·리워드 메뉴 추가 예정
- 미션 추가는 LNB에 직접 노출되지 않으며 **미션 목록에서만 진입** 가능

## 4. 기능 요구사항

### 4-1. 미션 관리 목록

> 등록된 미션을 1) **조회·ON/OFF 제어**, 2) **새 미션을 추가**할 수 있는 화면

#### 검색 조건

| 항목 | 설명 | 입력 형태 | 비고 |
|---|---|---|---|
| 상태 | 전체 / 예정 / 진행 / 완료 / 중단 | 드롭다운 |  |
| 목적 | 전체 / 전환 / 활성화 | 드롭다운 |  |
| 적용대상 유형 | 전체 / 일반기사(프렌즈) / NVP지점기사 / ~~벤더기사~~ | 드롭다운 |  |
| 미션명 | 미션명 키워드 검색 | 텍스트 |  |
| 검색 버튼 | - | 버튼 | 설정조건에 맞는 목록 조회 |
| 초기화 버튼 | - | 버튼 | 검색 조건 전체 초기화 클릭 시 전체 목록 조회 |

#### 미션 목록 항목

| 항목 | 설명 | 입력 형태 | 비고 |
|---|---|---|---|
| 미션명 | 미션 이름 | 텍스트(읽기전용) | 클릭 시 상세 화면 이동 |
| 목적 | 전환 / 활성화 | 텍스트(읽기전용) |  |
| 기간 | 시작일 ~ 종료일 | 텍스트(읽기전용) |  |
| 적용대상 유형 | 일반기사(프렌즈) / NVP지점기사 / ~~벤더기사~~ | 텍스트(읽기전용) | 벤더기사 P2(추후 추가) |
| 지역・ZONE | 미션 적용 지역/ZONE | 텍스트(읽기전용) | e.g. 미션적용지역: 서울시 강남구 서초동, ZONE: 원미ZONE, 중랑ZONE |
| 소진액 | 현재까지 지급된 총 금액 | 텍스트(읽기전용) |  |
| 예산 한도 | 설정된 총 예산 | 텍스트(읽기전용) |  |
| 남은 한도 | 잔여 예산 금액 + 잔여 비율 | 텍스트 + progress bar | e.g. 4,750,000원 / 59% 남음 |
| 상태 | 예정 / 진행 / 완료 / 중단 | 뱃지 | 예정: 회색 / 진행: 초록 / 완료: 회색 / 중단: 레드 |
| 관리 | ON / OFF 버튼 | 버튼 | 예정/진행 ON·OFF 전환 허용, 완료/중단(종료일 경과) "ON 불가" 표시. OFF 클릭 시 확인 모달 노출 |
| 총 건수 | 조회된 미션 수 | 텍스트(읽기전용) | 목록 우측 하단 표시 (e.g. 총 9건) |
| + 추가 | 미션 추가 화면 진입 | 버튼 | 목록 하단 좌측 노출 |

### 4-2. 미션 추가

> 미션의 조건을 설정하고 저장하는 화면

#### 기본 정보

| 항목 | 설명 | 필수 여부 | 입력 형태 | 동작 및 Validation |
|---|---|---|---|---|
| 미션명 | 미션 이름 | O | 텍스트 | 최대 50자 |
| 미션 설명 | 미션에 대한 설명 | X | 텍스트 | 최대 500자 |
| 목적 | 캠페인 운영 목적(전환/활성화) | O | Radio | - |
| 지급 구조 | 보상 지급 방식(단일/단계) | O | Radio | 전환 목적 시 미노출. 활성화 목적 시 선택 가능 |

#### 적용 조건

| 필드 | 설명 | 필수 여부 | 동작 및 Validation | 입력 형태 | 조건부 표시 |
|---|---|---|---|---|---|
| 기간 설정 | 캠페인 적용 기간 | O | 시작일 ≥ 오늘. 종료일 > 시작일 | 날짜 범위 | 공통 |
| 시간대 설정 | 적용 시간대 | X | 다중 구간 추가 가능. 시간 중복 불가. 미설정 시 전일 적용 | 시작~종료 + 추가(태그) | 활성화 목적 시 |
| 화주사 | 적용 화주사 범위 | X | 프랜차이즈/플랫폼 선택 시 세부 항목 드롭다운 노출. 미선택 시 전체 | 드롭다운 | 공통 |
| 미션 공개일 | 라이더에게 노출되는 날짜 | O | 시작일 이전이어야 함 | 날짜 | 공통 |
| 1인당 예산 | 캠페인 총 예산 | O | 0원 초과. 예산 소진 시 지급 중단 | 숫자(원) | 공통 |
| 가입 후 유효 일수 | 가입일로부터 미션 유효 기간 | O | 0 초과 정수 | 숫자(일) | 전환 목적 시 |
| 적용대상 유형 | 대상 라이더 유형 | O | 1개 이상 선택. 일반기사 선택 시 세그먼트 노출 | 체크박스 | 공통 |
| 라이더 활동 수준 | 활동 수준 분류 | O | 1개 이상 선택 | 체크박스 | 일반기사 선택 시 |
| ZONE | 적용 ZONE 선택 | O | 1개 이상 선택 | 다중 선택 | 항상 |

## Step 3. 달성 조건

### 운영 기준(선택 달성 조건)

| 필드 | 설명 | 필수 여부 | 동작 및 Validation | 입력 형태 |
|---|---|---|---|---|
| 휴무 조건 | 주당 최대 휴무 횟수 제한 | X | 0 이상 정수 | 숫자(회/주) |
| 취소건수 조건 | 기간 내 최대 취소 건수 제한 | X | 0 이상 정수 | 숫자(건) |

### 단계 지급 조건(지급 구조: 단계)

| 필드 | 설명 | 필수 여부 | 동작 및 Validation | 입력 형태 |
|---|---|---|---|---|
| 목표 건수 | 단계별 목표 배달 건수 | O | 0 초과 정수, 오름차순 | 숫자(건) |
| 지급 금액 | 단계별 지급 금액 | O | 0원 초과 | 숫자(원) |

- 최소 1단계, 최대 10단계
- 지급 기준: 미션 종료 다음날 카운팅 후 달성한 최고 단계 금액 1회 지급

### 단일 지급 조건(지급 구조: 단일)

| 필드 | 설명 | 필수 여부 | 동작 및 Validation | 입력 형태 |
|---|---|---|---|---|
| 1회 지급 금액 | 목표 달성 시 1회 지급 금액 | O | 0원 초과 | 숫자(원) |

- 지급 기준: 미션 종료 다음날 카운팅 후 목표 건수 달성 시 1회 지급
- 전환 목적: 가입 후 유효 일수 내 목표 건수 달성 시 1회 지급

## Step 4. 확인 및 저장

- 입력 항목 전체 요약 표시
- 저장 성공 시 목록으로 이동 + 성공 토스트
- 저장 실패 시 실패 사유 모달

## 4. 유효성 검증 규칙(요약)

| 규칙 | 조건 | 처리 |
|---|---|---|
| 필수 미입력 | 필수 항목 비어있음 | 저장 차단 + 하이라이트 |
| 시작일 오류 | 시작일 < 오늘 | 저장 차단 + 경고 |
| 공개일 오류 | 공개일 > 시작일 | 저장 차단 + 경고 |
| 단계 오름차순 | 이전 단계 ≥ 다음 단계 | 저장 차단 + 경고 |
| 예산 0 이하 | 1인당 예산 ≤ 0 | 저장 차단 + 경고 |
| 시간대 중복 | 시간대 구간 중복 | 저장 차단 + 경고 |
| 적용대상 미선택 | 적용대상 0개 | 저장 차단 + 경고 |
| 지역/ZONE 미선택 | 지역/ZONE 0개 | 저장 차단 + 경고 |
