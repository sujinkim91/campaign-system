## 1. 사용자 범위

| 사용자 | 설명 |
|---|---|
| 운영 관리자(3PL사업팀, 사업관리팀, BTF) | 인트라에서 `정적` 미션 캠페인을 등록·관리하는 내부 관리자 |

---

## 2. 캠페인 상태 정의

| 상태 | 설명 |
|---|---|
| 예정 | 등록 완료 후 시작일 전 |
| 진행 | 시작일 도달 시 자동 전환 |
| 완료 | 종료일 경과 시 자동 전환 |
| 중단 | 관리자 `OFF` 처리 |

### 상태 변경 기준

| 현재 상태 | 조건 | 변경 후 상태 | 비고 |
|---|---|---|---|
| 예정 | 시작일 도달 | 진행 | 시스템 자동 변경 |
| 진행 | 종료일 경과 | 완료 | 시스템 자동 변경 |
| 예정・진행 | 관리자 `OFF` | 중단 | 수동 |
| 중단 | 관리자 `ON` + 현재 시각 < 시작일 | 예정 | 아직 시작 전이므로 **예정**으로 복구 |
| 중단 | 관리자 `ON` + 시작일 ≤ 현재 시각 ≤ 종료일 | 진행 | 기간 내이므로 즉시 **진행**으로 복구 |
| 중단 | 종료일 경과 | 완료 | 자동, `ON` 불가 |

## 3. LNB 메뉴 구조

| depth 1 | depth 2 | depth 3 | 진입 방식 |
|---|---|---|---|
| 캠페인 관리 | 미션 관리 | 미션 목록 | LNB 클릭 |
| 캠페인 관리 | 미션 관리 | 미션 추가 | 미션 목록 내 + 추가 버튼 클릭 |

- 추후 프로모션·리워드 메뉴 추가 예정
- 미션 추가는 LNB에 직접 노출되지 않으며 **미션 목록에서만 진입** 가능

## 4. 기능 요구사항

### 4-1. 미션 관리 목록

#### 검색 조건

| 항목 | 설명 | 입력 형태 | 비고 |
|---|---|---|---|
| 상태 | 전체 / 예정 / 진행 / 완료 / 중단 | 드롭다운 |  |
| 목적 | 전체 / 전환 / 활성화 | 드롭다운 |  |
| 적용대상 유형 | 전체 / 일반기사(프렌즈) / NVP지점기사 | 드롭다운 |  |
| 미션명 | 미션명 키워드 검색 | 텍스트 |  |
| 검색 버튼 | - | 버튼 | 설정조건에 맞는 목록 조회 |
| 초기화 버튼 | - | 버튼 | 검색 조건 전체 초기화 클릭 시 전체 목록 조회 |

#### 미션 목록 항목

| 항목 | 설명 | 입력 형태 | 비고 |
|---|---|---|---|

| 목적 | 전환 / 활성화 | 텍스트(읽기전용) |  |
| 기간 | 시작일 ~ 종료일 | 텍스트(읽기전용) |  |

| 지역・ZONE | 미션 적용 지역/ZONE | 텍스트(읽기전용) | 실제 명칭만 표기; NVP **ZONE 전체** 적용 시 문구로 구분 |
| 소진액 | 현재까지 지급된 총 금액 | 텍스트(읽기전용) |  |

| 상태 | 예정 / 진행 / 완료 / 중단 | 뱃지 | 예정: 회색 / 진행: 초록 / 완료: 회색 / 중단: 레드 |

| + 추가 | 미션 추가 화면 진입 | 버튼 | 목록 하단 좌측 |

### 4-2. 미션 추가

> 미션의 조건을 설정하고 저장하는 화면

#### 기본 정보

| 항목 | 설명 | 필수 여부 | 입력 형태 | 동작 및 Validation |
|---|---|---|---|---|
| 미션명 | 미션 이름 | O | 텍스트 | 최대 50자, 공백 불가 |
| 미션 설명 | 미션에 대한 설명 | X | 텍스트 | 최대 500자 |
| 목적 | 캠페인 운영 목적(전환/활성화) | O | Radio | - |
| 지급 구조 | 보상 지급 방식(단일 목표형/단계형) | O | Radio | 전환 목적 시 미노출. 활성화 목적 시 선택 가능 |

#### 적용 조건 (표시·입력 순서)

| 필드 | 설명 | 필수 | 동작 및 Validation | 입력 형태 | 조건부 표시 |
|---|---|---|---|---|---|
| 미션 오픈일 | 미션 오픈 기준일 | O | **오픈일 ≥ 오늘** | 날짜 | 공통 |
| 기간 설정 | 캠페인 적용 기간 | O | 시작일 ≥ 오늘, 종료일 > 시작일 | 날짜 범위 | 공통 |
| 시간대 설정 | 적용 시간대 | X | 다중 구간 · 중복 불가 · 미설정 시 전일 | 시간 + 태그 | 활성화 목적 |
| 화주사 | 적용 화주사 범위 | X | 전체(기본)/로컬/프랜차이즈/플랫폼; 후자 선택 시 세부 입력 | 드롭다운+텍스트 | 공통 |

| 가입 후 유효 일수 | 가입일 기준 유효 기간 | O (전환) | 0 초과 정수 | 숫자(일) | 전환 목적 |
| 적용대상 유형 | 대상 라이더 유형 | O | 1개 이상 | 체크박스 | 공통 |
| 라이더 활동 수준 | 세부 분류 | X | 선택 | 체크박스 4종 | 일반기사(프렌즈) 선택 시 |

| 미션 대상 지역 | 픽업지 기준 | O (프렌즈) | 1개 이상 또는 전체지역 규칙 | 읍·면·동 | 일반기사(프렌즈) |

### 4-4. 저장·조회·수정 (요약)

- **저장 성공:** PRD에는 동일 화면 **조회 모드** 전환 및 **예산 한도·남은 한도** 노출 등이 포함될 수 있음. 정적 목업(`campaign-intra-mockup.html`)에서는 저장 후 **상세(조회) 화면으로 이동** + 토스트로 대응.
- **수정:** 예정은 전체 수정 허용; 진행·완료·중단 등은 제한(ON/OFF 등 세부는 원문 기준).
- **저장 실패:** 사유 모달

## 5. 유효성 검증 규칙(요약)

| 규칙 | 조건 | 처리 |
|---|---|---|
| 필수 미입력 | 필수 항목 비어 있음 | 저장 차단 + 하이라이트 |
| 미션 오픈일 | 오픈일 < 오늘 | 저장 차단 + 경고 |
| 시작일 | 시작일 < 오늘 | 저장 차단 + 경고 |
| 공개일 | 공개일 > 시작일 | 저장 차단 + 경고 |
| 단계 오름차순 | 이전 단계 ≥ 다음 단계 | 저장 차단 + 경고 |
| 예산 | 예산한도 ≤ 0 | 저장 차단 + 경고 |
| 시간대 중복 | 구간 중복 | 저장 차단 + 경고 |
| 적용대상 | 0개 | 저장 차단 + 경고 |
| ZONE | NVP 선택 시 전체 OFF이고 ZONE 0개 | 저장 차단 + 경고 |
| 지역 | 프렌즈 선택 시 활동/대상 지역 미충족 | 저장 차단 + 경고 |
