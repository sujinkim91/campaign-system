## 1. 배경 · 문제 · 목표

### 1.1 배경

내부 운영자가 미션 캠페인을 **등록·조회·ON/OFF**로 운영하고, 예산·기간·대상 조건을 일관되게 관리해야 한다.

- [TBD] NVP ZONE·프렌즈 지역 등 복수 조건의 저장·표시 규칙.

### 1.3 목표

1. **단일 화면 흐름**으로 목록 → 등록/조회/수정(허용 시)까지 완결.
2. 상태(예정/진행/완료/중단)와 **ON/OFF** 동작이 PRD·UI·백엔드에서 동일.
3. 예산(한도·소진·잔여)이 **저장 직후 조회 모드**에서 항상 확인 가능.

## 2. 범위

### 2.1 포함 (In scope)

- LNB: 캠페인 관리 > 미션 관리 > 미션 목록; **미션 추가는 목록의 +만**.
- 미션 목록: 필터, 컬럼, ON/OFF, 총 건수, 상세 진입.
- 미션 등록/조회/수정: 기본 정보, 적용 조건, 달성 조건, 운영 기준(휴무/취소), 저장 후 조회 모드·예산 패널.
- 상태 전이 및 OFF 복구 규칙(아래 §5).
- M캐시 지급·정원·휴무/취소 검증의 **제품 관점 요약**(시스템 상세는 연동 문서).

### 2.2 제외 (Out of scope)

- 라이더 앱 UI/알림 문구 최종 확정.
- Metabase/리포트 대시보드.
- 벤더기사 대상(P2 등 표기만 가능, 본 릴리스 필수 아님).

## 5. 도메인: 상태 · 전이

### 5.1 상태 정의

| 상태 | 설명 |
|---|---|
| 예정 | 등록 완료 후 시작일 전 |
| 진행 | 시작일 도달 시 자동 전환 |
| 완료 | 종료일 경과 시 자동 전환 |
| 중단 | 관리자 `OFF` 처리 |

### 5.2 전이 표

| 현재 | 조건 | 이후 | 비고 |
|---|---|---|---|
| 예정 | 시작일 도달 | 진행 | 자동 |
| 진행 | 종료일 경과 | 완료 | 자동 |
| 예정·진행 | 관리자 `OFF` | 중단 | 수동 |
| 중단 | `ON` + 현재 시각 < 시작일 | 예정 | 복구 |
| 중단 | `ON` + 시작일 ≤ 현재 ≤ 종료일 | 진행 | 복구 |
| 중단 | 종료일 경과 | 완료 | `ON` 불가 |

### 5.3 수용 기준 (예시)

- [ ] 종료일이 지난 미션은 목록에서 **ON 불가**이며 시도 시 사유가 노출된다.
- [ ] 중단 → ON 복구 시, 현재 시각이 위 표와 일치하는 상태로만 전이된다.

## 6. 기능 요구사항

### 6.1 미션 목록

**목적:** 등록 미션 **조회·ON/OFF** 및 **신규 등록 진입**.

#### 6.1.1 검색 조건

| 항목 | 설명 | 입력 | 비고 |
|---|---|---|---|
| 상태 | 전체/예정/진행/완료/중단 | 드롭다운 | |
| 목적 | 전체/전환/활성화 | 드롭다운 | |
| 적용대상 유형 | 전체/일반기사(프렌즈)/NVP지점기사/(벤더 P2) | 드롭다운 | |
| 미션명 | 키워드 | 텍스트 | |
| 검색 / 초기화 | 조회·조건 리셋 | 버튼 | 초기화 시 전체 목록 |

#### 6.1.2 목록 컬럼

| 항목 | 설명 | 비고 |
|---|---|---|
| 미션명 | 클릭 시 상세(조회/수정 흐름) | |
| 목적 | 전환/활성화 | |
| 기간 | 시작~종료 | |
| 적용대상 유형 | | |
| 지역·ZONE | 실제 명칭; NVP ZONE 전체 시 문구 구분 | |
| 소진액 | 누적 지급 | |
| 예산 한도 | 설정 총액 | |
| 남은 한도 | 금액 + % + progress | 예: `4,750,000원 / 59% 남음` |
| 상태 | 뱃지 색: 예정·완료 회색 / 진행 초록 / 중단 레드 | |
| 관리 | ON/OFF | 예정·진행만 토글; 완료·기간 종료 후 ON 불가; OFF 확인 모달 |
| 총 건수 | 우측 하단 | |
| + 추가 | 좌측 하단 | |

### 6.2 미션 등록 · 조회 · 수정

#### 6.2.1 기본 정보

| 항목 | 필수 | 입력 | Validation |
|---|---|---|---|
| 미션명 | O | 텍스트 | 최대 50자 |
| 미션 설명 | X | 텍스트 | 최대 500자 |
| 목적 | O | Radio 전환/활성화 | 구조·조건 분기 |
| 지급 구조 | O (활성화만 노출) | 단일/단계 | 전환 시 미노출 |

#### 6.2.2 적용 조건 (요약)

- 미션 오픈일: **오픈일 ≥ 오늘**
- 기간: **시작일 ≥ 오늘**, **종료일 > 시작일**
- 시간대: 활성화만, 다중 구간, 중복 불가, 미설정 시 전일
- 화주사: 전체/로컬/프랜차이즈/플랫폼 + 세부
- 미션 공개일: **공개일 ≤ 시작일**
- 예산한도: **0 초과**; 지급액 입력 시 `최대 참여 N명 · 1인당 N원` 안내(단계형은 최고 단계 금액 기준)
- 전환: 가입 후 유효 일수 **0 초과**
- 적용대상: 1개 이상
- 프렌즈: 활동 수준(선택), 활동 지역·미션 대상 지역(전체 토글 또는 선택)
- NVP: ZONE 전체 토글 또는 개별

#### 6.2.3 달성 조건

- **전환:** 유효 일수 내 목표 건수 달성 시 1회 지급(금액 필수).
- **활성화:** 단일형 / 단계형(오름차순, 최대 10단계 등 기존 규칙 유지).

#### 6.2.4 미션 운영 기준 (공통 노출)

| 항목 | 설명 | 필수 | Validation |
|---|---|---|---|
| 휴무 조건 | 기간 내 최대 휴무 횟수. **당일 부릉플러스 앱에서 오더받기 ON이 한 번도 없으면 1휴무로 산정** | X | 정수 ≥ 0 |
| 취소건수 조건 | 기간 내 취소 건수. **라이더 직접 배차 취소만** | X | 정수 ≥ 0 |

#### 6.2.5 저장 · 조회 모드

- 저장 성공: **동일 URL/화면에서 조회 모드** + 예산 한도·남은 한도(바) 표시.
- 저장 실패: 원인 모달.

#### 6.2.6 수정 정책 (현행 확정안)

| 상태 | 내용 수정 | ON/OFF |
|---|---|---|
| **예정** + **미션 오픈일 미도래** | **전 항목 허용** | 허용 |
| 예정이나 오픈일 도래 후 | [TBD] 전부 불가 vs 일부만 | [TBD] |
| 진행 / 완료 / 중단 | **내용 수정 제한** | 예정·진행만 토글 가능 등 [TBD와 표 5.2 정합] |

> **오픈일 미도래 정의(목업·API 공통):** 달력 기준 `mission_open_date > today`.

## 7. M캐시 · 정원 · 배치(요약)

- 참여 확정: 미션 기간 내 배송 1건 이상 수행 시 실시간 카운트.
- 최대 참여 인원: `floor(예산 ÷ 인당 최고 지급액)` (단계형은 최고 단계 금액).
- 정원 도달: 즉시 **진행 → 완료**.
- 지급: 미션 종료 **익일 배치** 후정산.

### 7.1 운영 기준 미충족 시

| 조건 | 기준 | 결과 |
|---|---|---|
| 휴무 | 기간 내 총 휴무 > 설정 | 지급 제외 |
| 취소 | 기간 내 총 취소 > 설정 | 지급 제외 |

## 8. 라이더 세그먼트(프렌즈)

- 산정 시점: 시작일 기준 **직전 2주**.
- 대상: 재직·부릉프렌즈·첫 수행 완료·직전 2주 배달 1건 이상 등 (운영 정의표 유지).
- 분류: SLA(고/저) × 물량(고/저) 4분류.
- 스냅샷 고정; 직전 2주 이력 없으면 **참여 대상 제외**.
